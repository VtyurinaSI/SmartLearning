# Decorator

Decorator (Декоратор) позволяет динамически добавлять объекту новые обязанности, оборачивая его в дополнительные объекты. В отличие от наследования, декораторы можно комбинировать в разных порядках, получая гибкую композицию поведения.

Структура паттерна:
- общий интерфейс компонента;
- конкретный компонент с базовой реализацией;
- базовый декоратор, который содержит ссылку на компонент;
- конкретные декораторы, расширяющие поведение.

Когда использовать:
- нужно добавлять функциональность без роста иерархии наследования;
- поведение должно комбинироваться в рантайме;
- требуется гибкая настройка функциональности (например, потоки ввода-вывода, форматирование, логирование).

Плюсы: гибкость и повторное использование, отсутствие «взрыва» подклассов.
Минусы: увеличивается число объектов и сложнее прослеживать цепочки оберток.
