# Command

Command (Команда) инкапсулирует запрос как объект, позволяя параметризовать клиентов, ставить команды в очередь, логировать или отменять операции. Вместо прямого вызова метода клиент создает объект-команду и передает его инициатору.

Участники паттерна:
- интерфейс команды с методом `Execute` (и иногда `Undo`);
- конкретные команды, которые вызывают действия у получателя (receiver);
- инициатор (invoker), который хранит команду и запускает выполнение;
- получатель, выполняющий реальную работу.

Когда использовать:
- нужно откладывать выполнение, ставить команды в очередь;
- требуется история и отмена операций;
- важно отделить отправителя запроса от получателя.

Плюсы: хорошая расширяемость, удобство логирования, возможность макрокоманд.
Минусы: увеличивается количество классов, иногда усложняется структура приложения.
